## How the Dev Role Override Works
- `/portal-select-v6` calls `user.update({ unsafeMetadata: { devRoleOverride: "role_name" } })`
- Each v6 portal reads `(clerkUser.unsafeMetadata as any)?.devRoleOverride` before `publicMetadata.role`
- This lets you test any role's RBAC view without re-logging-in or changing your real Clerk role
- `unsafeMetadata` is client-writable; `publicMetadata` is server-only (set via webhook or Clerk dashboard)
