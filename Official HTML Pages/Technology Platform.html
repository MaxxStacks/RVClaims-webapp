Access Control

Four Permission Tiers. Zero Data Leakage.

DS360 uses role-based access control (RBAC) with four distinct permission tiers. Each role sees only what it needs to see — nothing more. This is not a simple admin/user toggle. It is a granular permission system that controls access at the feature, data, and action level.

Operator / Super Admin

Full platform access — all dealers, all claims, all configuration, billing, invoicing, and system administration

Operator Staff

Full claims access across all dealers — no billing, invoicing, or admin configuration

Dealer Owner

Full dealership control — claims, units, billing, staff management, F&I, marketplace, and reporting for their dealership only

Dealer Staff

Operational access only — claims and units for their dealership. No billing, no admin, no staff management

Data Architecture

Multi-Tenant Isolation — Your Data Is Yours

Every dealer on DS360 operates in an isolated workspace. Dealer A cannot see Dealer B's claims, units, clients, billing, or any other data — even though both run on the same platform. This is not folder-level separation. It is database-level tenant isolation built into the core architecture.

Database-Level Isolation

Tenant boundaries enforced at the data layer — not through application-level filtering

3-Layer Model

Operator → Dealer → Client. Each layer sees only its own data and the data its children generate

Operator Is Invisible

The client sees only the dealer's brand. DS360 operates behind the scenes — the operator layer is never exposed to the end user

Did You Know?

Most dealer management platforms use application-level data separation — meaning all dealer data sits in the same database tables, separated only by software logic. If that logic fails, data leaks. DS360 uses tenant-level isolation — each dealer's data is structurally separated at the database level. Even if the application layer had a bug, cross-tenant data exposure is architecturally prevented.
